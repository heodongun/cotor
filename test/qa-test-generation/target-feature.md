# QA Test Generation Target

## 대상 기능

주문 생성 서비스의 입력 검증과 중복 제출 방지 로직을 테스트 대상으로 삼습니다.

## 비즈니스 규칙

1. `customerId` 는 비어 있으면 안 됩니다.
2. 주문 항목은 최소 1개 이상이어야 합니다.
3. 각 주문 항목의 수량은 1 이상이어야 합니다.
4. 동일한 `requestId` 로 중복 요청이 오면 새 주문을 만들지 않고 기존 결과를 반환해야 합니다.
5. 저장 실패가 발생하면 감사 로그가 남아야 합니다.

## 기대 테스트 관점

- 입력 검증 실패 케이스
- 멱등성(`requestId`) 처리
- 저장소 예외 발생 시 감사 로그 호출
- 정상 주문 생성 시 총 수량/주문 ID 반환
