# Paperclip 대비 벤치마크 리포트

## 목적

HEO-74의 목표는 Cotor 저장소 안에 남아 있는 실행 결과를 기준으로 현재 성능/안정성 신호를 정리하고, Paperclip 대비 비교 가능 여부를 명확히 문서화하는 것입니다.

## 범위와 한계

- 이 리포트는 이 저장소에 체크인된 산출물만 사용합니다.
- `Paperclip` 문자열, Paperclip 실행 로그, Paperclip 측정치, Paperclip 설정 파일은 현재 저장소에서 확인되지 않았습니다.
- 따라서 이 문서는 "동일 조건의 정면 비교 결과"가 아니라, Cotor의 현재 벤치마크 근거와 Paperclip 기준선 부재를 함께 정리한 비교 준비 리포트입니다.

## 사용한 근거 자료

1. `.cotor/stats/conditional.json`
2. `.cotor/stats/looping.json`
3. `test-results/README.md`
4. `test/results/compare-solutions-result.md`
5. `test-claude/EXPERIMENT_RESULTS.md`

grep -RIn --exclude-dir=.git --exclude-dir=node_modules --exclude-dir=build --exclude-dir=.gradle "Paperclip" .

## 핵심 수치 요약

### 1. 경량 오케스트레이션 실행 통계

`.cotor/stats/*.json` 기준:

| 파이프라인 | 실행 수 | 무실패 실행 | 평균 시간 | 중앙값 | 최근 10회 평균 | 최소/최대 |
|---|---:|---:|---:|---:|---:|---:|
| `conditional` | 29 | 29/29 | 33.31ms | 24.00ms | 24.50ms | 15ms / 85ms |
| `looping` | 29 | 29/29 | 7.72ms | 5.00ms | 4.70ms | 3ms / 21ms |

해석:

- 조건 분기와 루프 제어 자체의 오버헤드는 밀리초 단위로 유지됩니다.
- 체크인된 통계 범위에서는 두 파이프라인 모두 실패 기록 없이 누적되었습니다.

### 2. 실제 CLI 기능 검증

`test-results/README.md` 기준:

| 항목 | 결과 |
|---|---|
| 전체 검증 항목 | 13개 |
| 성공/실패 | 13 성공 / 0 실패 |
| 설치 시간 | 약 4초 |
| 파이프라인 실행 | 약 106ms |
| 템플릿 생성 | 1초 미만 |
| 검증 명령 | 1초 미만 |

해석:

- 문서화된 로컬 라이브 테스트에서는 설치, 검증, 템플릿 생성, 단일 파이프라인 실행이 모두 통과했습니다.
- 최소한의 사용성/실행 경로는 안정적으로 재현된 상태입니다.

### 3. 멀티 AI 비교 실행

`test/results/compare-solutions-result.md` 기준:

| 항목 | 측정치 |
|---|---|
| 총 소요 시간 | 48.2초 |
| 성공률 | 3/3 (100%) |
| Claude | 28.4초 |
| Codex | 7.5초 |
| Gemini | 12.3초 |

해석:

- 병렬 비교 시나리오 자체는 성공적으로 수행되었습니다.
- 동일 과제를 여러 에이전트로 비교하는 템플릿/결과 수집 흐름이 실제 결과물로 남아 있습니다.

### 4. 장시간 실험에서 드러난 한계

`test-claude/EXPERIMENT_RESULTS.md` 기준:

| 실험 | 결과 | 시간 | 메모 |
|---|---|---:|---|
| `01-basic` | 성공 | 17.1초 | 단일 AI 호출 |
| `02-parallel` | 부분 성공 | 55.7초 | Gemini API 에러로 50% 성공률 |

해석:

- Cotor는 기본/병렬 실행 자체는 수행했지만, 외부 AI 실패 시 fallback 부재가 병목으로 드러났습니다.
- 즉, 오케스트레이션 오버헤드는 낮지만 외부 모델 신뢰도까지 포함한 end-to-end 안정성은 에이전트 장애 처리 전략에 크게 좌우됩니다.

## Paperclip 대비 비교 표

현재 저장소 기준으로 검증 가능한 비교는 아래 수준까지입니다.

| 비교 항목 | Cotor 근거 | Paperclip 근거 | 판정 |
|---|---|---|---|
| 체크인된 실행 통계 존재 여부 | `.cotor/stats/*.json` 29회 누적 통계 확인 | 저장소 내 근거 없음 | Cotor만 검증 가능 |
| 라이브 테스트 리포트 | `test-results/README.md` 13/13 성공 | 저장소 내 근거 없음 | Cotor만 검증 가능 |
| 멀티 AI 비교 실행 결과 | `test/results/compare-solutions-result.md` 존재 | 저장소 내 근거 없음 | Cotor만 검증 가능 |
| 장애 복구 전략 검증 | Gemini 실패 사례와 개선 필요 문서화 | 저장소 내 근거 없음 | 정량 비교 불가 |
| 동일 워크로드 head-to-head 시간 비교 | 부분 근거만 존재 | 원본 기준선 없음 | 비교 불가 |

## 결론

이 저장소만 기준으로 하면 다음은 분명합니다.

1. Cotor는 체크인된 로컬 증거 기준으로 경량 오케스트레이션 경로에서 낮은 실행 오버헤드와 높은 반복 성공률을 보입니다.
2. 멀티 AI 비교, 라이브 CLI 테스트, 조건/루프 통계까지는 실제 산출물이 남아 있어 재검토 가능합니다.
3. 그러나 Paperclip 원본 측정치가 마이그레이션되지 않아, HEO-74를 완전한 정면 벤치마크로 닫기에는 비교 기준선이 비어 있습니다.

따라서 이 문서의 실질적 산출물은 다음 두 가지입니다.

- Cotor 쪽 벤치마크 근거를 한 곳에 정리한 저장소 내 기준 리포트
- Paperclip 대비 수치 비교가 왜 아직 불완전한지 설명하는 증거 기반 gap 문서
