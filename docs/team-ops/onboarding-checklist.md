# Onboarding Checklist (Day 0-14)

## Day 0 (Access & Setup)

- [ ] Repository read/write access granted.
- [ ] Local development environment boots successfully.
- [ ] CLI smoke test succeeds (`cotor --short`, `cotor doctor`).
- [ ] Team communication channels joined.
- [ ] Issue tracker and PR workflow reviewed.

## Day 1-3 (Codebase Familiarization)

- [ ] Read architecture and features docs.
- [ ] Run at least one sample pipeline.
- [ ] Understand branching, commit, and PR conventions.
- [ ] Review test strategy and CI expectations.

## Day 4-7 (Guided Contribution)

- [ ] Pick a starter issue with maintainer support.
- [ ] Open first PR with a clear test log.
- [ ] Practice review response loop (update, re-test, summarize).

## Day 8-14 (Independent Ownership)

- [ ] Lead one scoped task end-to-end.
- [ ] Handle one triage or documentation update independently.
- [ ] Create handoff notes for the next contributor.

## Exit Criteria

- [ ] Can run, validate, and explain a standard pipeline flow.
- [ ] Can open a high-quality PR with clear testing evidence.
- [ ] Can use the operating cadence template without assistance.
