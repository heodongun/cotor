# 역할별 운영 템플릿 (초안)

아래 템플릿은 이슈 스레드, PR 설명, 워크패드에 복사해서 쓰는 체크리스트입니다.

## 1) 기여자 템플릿

### 컨텍스트
- 이슈:
- 브랜치:
- 범위(포함/제외):

### 일일 작업 계획
- [ ] 수용 기준(AC) 재확인
- [ ] 변경 범위가 이슈와 일치하는지 점검
- [ ] 커밋 전 로컬 검증 수행
- [ ] 스레드에 검증 근거 기록

### 전달(완료) 체크리스트
- [ ] 코드/문서 변경이 요청 범위와 일치
- [ ] 사용자 영향 변경점 문서화
- [ ] 리스크/롤백 메모 작성

---

## 2) 리뷰어/메인테이너 템플릿

### 리뷰 포인트
- [ ] 변경 범위가 이슈 의도와 일치
- [ ] 핵심 경로 검증(테스트/체크) 여부 확인
- [ ] 문서/UX 변경이 진입점에서 발견 가능
- [ ] 커밋/PR 메시지에 변경 이유와 내용이 명확

### 병합 게이트
- [ ] CI 통과(예외 시 근거 문서화)
- [ ] 블로킹 코멘트 해소
- [ ] 릴리스 노트/체인지로그 영향 검토

---

## 3) 릴리스 담당자 템플릿

### 릴리스 전
- [ ] 릴리스 대상 커밋/태그 확인
- [ ] 체인지로그 항목 누락 여부 점검
- [ ] 핵심 명령/플로우 스모크 체크

### 릴리스 수행
- [ ] 아티팩트 배포
- [ ] 변경 사항/알려진 제한 공유
- [ ] 릴리스 직후 이슈 모니터링

---

## 4) 문서 오너 템플릿

### 문서 품질 게이트
- [ ] 신규 문서가 루트 README 및 docs 랜딩에서 노출됨
- [ ] 한/영 문서 정합성 고려
- [ ] 예제 명령/경로 유효성 점검
- [ ] 오래된 참조 제거 또는 상태 표시

### 인수인계
- [ ] 기여자를 위한 요약 추가
- [ ] 후속 문서 부채 항목 기록
