# 메인터이너 템플릿

여러 기여자가 동시에 작업하는 저장소를 배포 가능 상태로 유지할 때 사용하는 템플릿입니다.

## 목적

- 온보딩, 문서, 릴리즈 흐름을 서로 맞물리게 유지합니다.
- 상위 문서, 상세 가이드, 실제 운영 흐름 사이의 드리프트를 잡습니다.
- 무엇이 왜 바뀌었는지 깔끔한 기록을 남깁니다.

## 주간 체크리스트

- 상위 문서의 진입점이 깨졌거나 빠진 곳이 없는지 훑습니다.
- 열린 티켓/PR에서 반복되는 운영 공백이 있는지 확인합니다.
- 반복되는 리뷰 코멘트를 문서나 템플릿으로 흡수합니다.
- 릴리즈 노트나 업그레이드 가이드 갱신이 필요한지 점검합니다.

## 릴리즈/변경 체크리스트

- 상위 README, 문서 랜딩 페이지, 기능 문서가 서로 일치하는지 확인합니다.
- 새 운영 규칙은 한 번만 자세히 쓰고 나머지 문서에서는 링크로 연결합니다.
- 배포 전 복사 템플릿이 현재 워크플로우와 맞는지 검증합니다.

## 복사 템플릿

```md
Release area: <docs / app / workflow>
Changed because: <reason>
Entry points updated: <README / docs index / other>
Validation: <commands or walkthrough>
Follow-up needed: <none or issue link>
```
