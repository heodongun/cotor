# 기능 담당자 템플릿

한 사람이 티켓의 전달 루프 전체를 맡을 때 사용하는 템플릿입니다.

## 목적

- 범위를 작고 명확하게 유지합니다.
- 변경 전후 근거를 남깁니다.
- 브랜치, workpad, PR을 바로 리뷰 가능한 상태로 정리합니다.

## 시작 체크리스트

- 이슈 상태, acceptance criteria, validation 요구사항을 확인합니다.
- 명령, 스크린샷, 또는 문서 공백처럼 현재 상태를 재현 가능한 형태로 기록합니다.
- 수정 전에 `origin/master`와 동기화합니다.
- 티켓을 해결하는 최소 파일 집합을 정합니다.

## 진행 체크리스트

- 구현 전에 workpad 계획을 갱신합니다.
- 가장 작은 변경으로 해결합니다.
- 의미 있는 마일스톤마다 검증합니다.
- 발견한 내용을 로컬 메모에 두지 말고 workpad에 반영합니다.
- 문서 발견성이 바뀌었다면 상위 README/인덱스도 다시 확인합니다.

## 종료 체크리스트

- 최종 diff 기준 검증 명령이 모두 통과합니다.
- workpad에 최종 계획, 메모, 검증 근거가 반영됩니다.
- PR 요약에 사용자/리뷰어 영향, 변경 파일, 검증 근거가 포함됩니다.
