# 온보딩 체크리스트 (Day 0-14)

## Day 0 (권한/환경 준비)

- [ ] 저장소 읽기/쓰기 권한 부여 완료
- [ ] 로컬 개발 환경 정상 부팅 확인
- [ ] CLI 스모크 테스트 성공 (`cotor --short`, `cotor doctor`)
- [ ] 팀 커뮤니케이션 채널 참여 완료
- [ ] 이슈/PR 기본 워크플로 확인

## Day 1-3 (코드베이스 이해)

- [ ] 아키텍처/기능 문서 정독
- [ ] 샘플 파이프라인 최소 1회 실행
- [ ] 브랜치/커밋/PR 규칙 이해
- [ ] 테스트 전략 및 CI 기대사항 파악

## Day 4-7 (가이드 기여)

- [ ] 메인테이너 지원 하에 스타터 이슈 선택
- [ ] 테스트 로그를 포함한 첫 PR 오픈
- [ ] 리뷰 대응 루프(수정→재검증→요약) 수행

## Day 8-14 (독립 수행)

- [ ] 범위가 명확한 작업 1건 end-to-end 리드
- [ ] 트리아지 또는 문서 업데이트 1건 독립 수행
- [ ] 다음 기여자를 위한 handoff 노트 작성

## 종료 기준

- [ ] 표준 파이프라인 실행/검증 흐름을 설명 가능
- [ ] 테스트 근거가 명확한 PR을 독립적으로 작성 가능
- [ ] 운영 cadence 템플릿을 보조 없이 활용 가능
