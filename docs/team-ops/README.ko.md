# 팀 운영 온보딩 패키지 (초안)

이 패키지는 Cotor 저장소에서 기여자와 메인테이너가 공통 기준으로 시작할 수 있도록 만든 저장소 전용 온보딩 시작점입니다.

## 대상

- 기능 개발/버그 수정에 합류한 신규 기여자
- 릴리스, 이슈 트리아지, 품질 게이트를 담당하는 메인테이너
- 운영 기준을 공유해야 하는 협업자

## 구성

- `templates.md`: 역할별 운영 템플릿(영문)
- `templates.ko.md`: 역할별 운영 템플릿(국문)

## 권장 온보딩 흐름

1. `docs/README.md`, `docs/INDEX.md`로 전체 문서 지형을 파악합니다.
2. `ARCHITECTURE.md`, `QUICK_START.md`를 훑어 실행 경로를 이해합니다.
3. `team-ops/templates.ko.md`에서 본인 역할 템플릿을 복사합니다.
4. 담당자/범위/체크리스트를 채우고 작업 이슈·PR 링크를 연결합니다.
5. 이슈 스레드 또는 브랜치 워크패드에 유지하며 매일 갱신합니다.

## 초안에서 포함한 역할

- 기여자(기능/버그 구현)
- 리뷰어/메인테이너(리뷰, 병합, 안정성 보장)
- 릴리스 담당자(변경 이력, 패키징, 스모크 체크)
- 문서 오너(문서 품질 및 탐색성)

## 상태

- 본 문서는 COT-13 / HEO-77 범위의 초안입니다.
- 팀 피드백을 반영해 템플릿 항목을 점진적으로 구체화할 예정입니다.
