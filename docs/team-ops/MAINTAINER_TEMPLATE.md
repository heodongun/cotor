# Maintainer Template

Use this when you are keeping the repository shippable across multiple contributors.

## Mission

- Keep onboarding, docs, and release flow coherent.
- Catch drift between top-level docs, detailed guides, and active workflows.
- Leave a clean record of what changed and why.

## Weekly Checklist

- Scan top-level docs for broken or missing entry points.
- Review open tickets/PRs for recurring workflow gaps.
- Consolidate repeated reviewer comments into docs or templates.
- Check whether release notes or upgrade guides need updates.

## Release/Change Checklist

- Confirm top-level README, docs landing pages, and feature docs still agree.
- Ensure new operational expectations are written once and linked everywhere else.
- Validate copy/paste templates against the current workflow before sharing them.

## Copy/Paste Template

```md
Release area: <docs / app / workflow>
Changed because: <reason>
Entry points updated: <README / docs index / other>
Validation: <commands or walkthrough>
Follow-up needed: <none or issue link>
```
