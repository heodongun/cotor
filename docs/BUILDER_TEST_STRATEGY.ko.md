# Builder 테스트 전략: 목표 `test`

원문: [BUILDER_TEST_STRATEGY.md](BUILDER_TEST_STRATEGY.md)

이 문서는 Builder(`gemini`) 에이전트에 할당된 `test` 목표의 성공 조건과 실행 계획을 한국어로 정리한 동반 문서입니다.

## 목표: `test`

**설명**: `"안녕"`

## 역할: Builder

**초점**: 프런트엔드 동작, 상호작용 품질, 시각적 완성도

## 성공 조건

### 1. 상호작용 품질

- `"안녕"` 인사에 자연스럽고 명확하게 응답해야 합니다.
- CLI 출력이나 보고서가 구조적이고 읽기 쉬워야 합니다.

### 2. 구현 및 시각적 완성도

- `gemini` 에이전트를 쓰는 `pipelines/gemini-test.yaml` 파이프라인이 있어야 합니다.
- 출력은 구조화된 Hello World 형태여야 하고, 표나 ASCII 장식 같은 시각 요소를 포함할 수 있습니다.
- `cotor.yaml`에 새 파이프라인과 에이전트 설정이 반영되어야 합니다.

### 3. 실행 및 검증

- `cotor validate`가 통과해야 합니다.
- `cotor run`이 정상 동작해야 합니다.
- 출력 결과의 시각적/기능적 품질을 캡처해서 확인해야 합니다.

### 4. 리스크 평가

- API 키나 CLI 설치 여부 같은 실행 환경 리스크를 확인해야 합니다.
- 불가능한 경우 fallback 또는 명확한 오류 메시지가 있어야 합니다.

## 실행 계획

1. `gemini` 플러그인이 실제 사용 가능한지 확인합니다.
2. `pipelines/gemini-test.yaml`을 생성합니다.
3. `cotor.yaml`에 에이전트와 import를 추가합니다.
4. 파이프라인을 검증하고 실행합니다.
5. 결과를 정리해 인계합니다.
