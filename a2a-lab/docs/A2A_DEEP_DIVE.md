# A2A Deep Dive (실습 중심)

## 1) A2A란?
A2A(Agent-to-Agent)는 **여러 에이전트가 역할을 분담하고, 결과를 서로 전달하며, 하나의 업무를 완성**하는 방식입니다.

핵심 아이디어:
- 단일 거대 에이전트 대신 역할 분리
- 결과물 기반 협업(메시지/산출물 전달)
- 워크플로우로 재현 가능하게 운영

---

## 2) 왜 A2A가 필요한가?
단일 에이전트 방식은 간단하지만, 실무에서는 다음 문제가 생깁니다.

- 컨텍스트 과부하: 한 에이전트가 모든 맥락을 떠안음
- 책임 불명확: 조사/계획/구현/검증의 경계가 흐림
- 재사용 어려움: 특정 역할 프롬프트를 다른 작업에 재활용하기 어려움

A2A는 이를 아래처럼 바꿉니다.
- Researcher: 자료 조사
- Planner: 계획 수립
- Coder: 구현
- Tester: 검증
- Reviewer: 품질 점검

---

## 3) 이 실습 프로젝트의 구조

### 서버 (`server.js`)
- `GET /api/agents`: 사용 가능한 에이전트 목록 제공
- `POST /api/workflows/execute`: 워크플로우 실행

실행 로직:
1. 노드/엣지 검증
2. 위상 정렬(사이클 감지)
3. 순서대로 각 노드 실행
4. 이전 노드 결과를 `context`로 전달
5. 최종 `results` + `logs` 반환

### 클라이언트 (`public/index.html`)
- 에이전트 카드 표시
- 노드 ID/에이전트/입력/의존성 입력
- 실행 결과 JSON 시각화

---

## 4) 워크플로우 모델(DAG)
워크플로우는 Directed Acyclic Graph(DAG)로 표현합니다.

- Node: 하나의 에이전트 실행 단위
- Edge: 의존 관계(from → to)

예시:
- `n1(researcher)` → `n2(planner)` → `n3(coder)` → `n4(tester)`
- `n2`, `n4` → `n5(reviewer)`

이 구조의 장점:
- 실행 순서가 명확
- 병렬화 가능 지점 발견 용이
- 오류 발생 시 실패 노드 기준으로 재시작 전략 수립 가능

---

## 5) A2A 설계에서 중요한 포인트

### (1) Agent Contract
각 에이전트는 아래 계약을 가져야 합니다.
- 입력 스키마
- 출력 스키마
- 실패 시 에러 형식

실무에서는 JSON Schema/Pydantic/Zod로 강제하는 걸 추천.

### (2) Context 최소화
무조건 이전 결과를 전부 넘기지 말고,
- 필요한 요약만 전달
- 토큰/비용 절감
- 프롬프트 안정성 향상

### (3) Observability
최소한 아래는 남겨야 합니다.
- workflow_id
- node_id
- agent_id
- 시작/종료 시간
- 토큰/비용(LLM 연동 시)
- 실패 원인

### (4) Idempotency & Retry
재시도 시 중복 실행 부작용을 줄이려면
- 외부 시스템 쓰기 작업에 idempotency key 사용
- 노드 단위 재시도 정책 분리(즉시/지수백오프)

### (5) Human-in-the-loop
중요 승인 지점(배포/결제/데이터 삭제)은 자동 실행 금지.
- 승인 노드 추가
- Slack/Email 승인 버튼 연동

---

## 6) 지금 프로젝트를 실전형으로 키우는 방법
1. 실제 LLM 연결
   - 각 에이전트의 `run()` 내부를 API 호출로 교체
2. 워크플로우 저장소
   - SQLite/Postgres에 workflow template 저장
3. 시각 워크플로우 편집기
   - React Flow 같은 라이브러리로 노드-엣지 편집
4. 병렬 실행 엔진
   - indegree 0 노드들을 동시 처리(Promise.all)
5. 보안/권한
   - 실행 권한 분리, 감사 로그, 비밀키 vault 적용

---

## 8) 한 줄 결론
A2A는 “똑똑한 단일 에이전트”보다 **“운영 가능한 협업 시스템”**을 만드는 접근입니다.

이 실습 앱은 그 최소 골격(역할 분리 + DAG + 실행 로그)을 제공하며,
여기에 LLM/저장소/승인흐름을 붙이면 팀 단위 자동화 플랫폼으로 확장할 수 있습니다.
